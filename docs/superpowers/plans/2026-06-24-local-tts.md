# Local TTS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a lightweight offline TTS module and expose it as a reusable voice assistant capability.

**Architecture:** Create a standalone `components/local_tts` component that turns UTF-8 text into 16 kHz mono PCM with a compact built-in synthesizer. Add a `voice_assistant_tts_t` capability and `voice_assistant_speak_text()` bridge that plays synthesized PCM through the existing audio port.

**Tech Stack:** C11, ESP-IDF component CMake, host CTest tests, existing voice assistant SDK, ES8311 PCM playback hook.

---

## File Structure

- Create `components/local_tts/include/local_tts.h`: public TTS API and config/result structs.
- Create `components/local_tts/src/local_tts.c`: compact built-in UTF-8 to PCM synthesizer.
- Create `components/local_tts/CMakeLists.txt`: ESP-IDF component registration.
- Modify `components/voice_assistant_sdk/include/voice_assistant.h`: add TTS capability type and speak API.
- Modify `components/voice_assistant_sdk/src/voice_internal.h`: declare internal speak helper if needed.
- Create `components/voice_assistant_sdk/src/voice_tts.c`: synthesize text and play PCM through `audio_port.play_pcm`.
- Modify `components/voice_assistant_sdk/CMakeLists.txt`: compile `voice_tts.c` and require `local_tts`.
- Modify `components/voice_assistant_sdk/Kconfig`: add `VOICE_ASSISTANT_LOCAL_TTS`.
- Modify `main/main.c`: configure local TTS and speak assistant text events.
- Modify `tests/CMakeLists.txt`: compile local TTS tests and link local TTS into SDK tests.
- Create `tests/test_local_tts.c`: unit tests for the standalone module.
- Modify `tests/test_voice_assistant_sdk.c`: unit tests for the voice assistant TTS capability.
